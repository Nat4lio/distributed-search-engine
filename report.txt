Projeto Googol – Meta 2

Sistemas Distribuídos 2025/2026

1. Objetivos do Projeto

O objetivo desta segunda meta consiste no desenvolvimento de uma interface Web para o motor de pesquisa Googol, previamente implementado na Meta 1. Pretende-se disponibilizar as mesmas funcionalidades da aplicação distribuída original através de um navegador Web, mantendo a interoperabilidade com os componentes existentes via RPC/RMI.

Adicionalmente, esta meta introduz:

Programação Web segundo o modelo MVC

Comunicação assíncrona com WebSockets

Integração com serviços REST externos

Atualização de estatísticas em tempo real

2. Visão Geral

O sistema Googol é um motor de pesquisa distribuído que indexa páginas Web automaticamente e permite aos utilizadores realizar pesquisas por termos. A arquitetura baseia-se em múltiplos componentes distribuídos:

Gateway: coordena pesquisas e estatísticas

Barrels: armazenam o índice invertido

Downloaders: efetuam crawling e indexação

Fila de URLs: gere URLs pendentes

Servidor Web (Meta 2): interface Web para utilizadores

A aplicação Web desenvolvida nesta meta atua como cliente do Gateway via RMI, garantindo que utilizadores Web e desktop acedem à mesma informação.

3. Arquitetura do Sistema
3.1 Visão Geral da Arquitetura

A arquitetura segue um modelo cliente–servidor distribuído, onde:

O Servidor Web comunica com o Gateway via RPC/RMI

O Gateway comunica com os Barrels

Os Downloaders alimentam o sistema através da fila de URLs

Os Browsers comunicam via HTTP e WebSockets com o Servidor Web

Os diferentes componentes do sistema (Gateway, Barrels, Downloaders e Servidor Web) podem ser executados em processos e máquinas distintas, comunicando exclusivamente através de mecanismos de rede, cumprindo os requisitos de um sistema distribuído.

4. Interface Web (MVC)

A aplicação Web segue a arquitetura Model–View–Controller.

4.1 Controllers

IndexController
Página inicial da aplicação.

SearchController
Processa pesquisas por termos, apresenta resultados paginados (10 por página) e ordenados por relevância.

InboundController
Permite a submissão manual de URLs para indexação.

StatsWebController
Apresenta estatísticas globais do sistema.

RestIntegrationController
Gere pedidos relacionados com integrações REST externas.

4.2 Models

SearchResult
Representa um resultado de pesquisa (URL, título e citação).

PageInfo
Contém metadados sobre uma página indexada.

4.3 Services

StatsService
Fornece estatísticas agregadas (top pesquisas, Barrels ativos e tempos médios).

5. Funcionalidades Implementadas
5.1 Indexação de URLs

Submissão manual de URLs via interface Web

Indexação automática e recursiva através dos Downloaders

Utilização de fila de URLs para crawling iterativo

5.2 Pesquisa

Pesquisa por múltiplos termos

Resultados apenas com páginas que contêm todos os termos

Paginação de resultados (10 por página)

5.3 Ordenação por Relevância

Resultados ordenados com base no número de ligações recebidas

Barrels mantêm informação de backlinks por URL

5.4 Consulta de Backlinks

Possibilidade de visualizar páginas que apontam para um determinado URL
