2. Arquitetura de Software
	
Gateway

	Coordenador central.
	
	Recebe queries dos clientes.

	Distribui pesquisas entre os barrels (round-robin).

	Indexa páginas enviadas pelo downloader.

	Tolerante a falhas dos barrels ativos.

	Mantém cache das pesquisas mais recentes.

Barrel	

	Armazena o índice invertido palavra → conjunto de URLs.

	Mantém informações de páginas (PageInfo) e links de entrada (inboundLinks).

	Permite atualização incremental de índice (putIndex) e consulta (searchUrls).

	Pode ser sincronizado com outros barrels ao ser registado no Gateway.

Downloader / Indexador

	Baixa páginas web.

	Extrai texto, título, snippet e links de saída.

	Constroi mapa invertido (palavra → URL) e envia para os barrels via Gateway.

	Suporta indexação paralela usando ForkJoinPool.

	Detecta dinamicamente novos barrels.

Client

	Interface para pesquisa e obtenção de informações.

	Não precisa saber detalhes de qual barrel serve a query.



Downloader baixa páginas e envia PageInfo + índice parcial para o Gateway.

Gateway distribui o índice pelos barrels ativos.

Clientes enviam queries para o Gateway.

Gateway consulta os barrels (round-robin), agrega resultados, ordena e devolve ao cliente.

Gateway mantém cache de queries recentes.



3. Funcionamento do RPC/RMI


Interfaces remotas:

	BarrelInterface → métodos como putIndex, searchUrls, getPageInfo, inboundLinks.

	GatewayInterface → métodos como search, indexPage, inboundLinks, getStats.

Componentes:

	Cada Barrel implementa UnicastRemoteObject e se registra no RMI Registry.

	Gateway implementa UnicastRemoteObject, é o ponto de contacto do cliente.

	Comunicação entre cliente ↔ Gateway e Gateway ↔ Barrel via RMI.
	
Mecanismos adicionais:

	ping() usado para verificar se barrels estão ativos.

	Synchronização de novos barrels com índices existentes (getFullIndex + putIndex).



4. Replicação (Reliable Multicast)

Quando um novo barrel é registado, o Gateway sincroniza-o com o último barrel ativo:

Gateway chama getFullIndex no barrel existente.

Copia índices invertidos e páginas.

Chama putIndex no novo barrel.

Garante que todos os barrels têm cópias consistentes do índice.

Permite tolerância a falhas: se um barrel cair, os restantes continuam a servir queries.
